Maebot: Mobile Robot

Maze Runner

The objective of this project was to program a rover (APRIL Lab Maebot), equiped with a laser radar (Lidar), to navigate a maze towards a goal point. Using Similtaneous Location and Mapping (SLAM), we could locate the robot relative to the walls of the maze and create a map of our measurements. The map of the course was built on the fly, so the robot had to use a path planner to decide where to travel based on new incoming data. Lastly, SLAM played a role in trajectory correction, as the robot's movement was not perfect due to dirt on the ground or misaligned wheels.

Motor Control and Odometry

The Maebot is driven by two DC motors with encoders which is controlled by two PID velocity controllers. The Maebot can transverse a path by odometry but will accumulate error due to intrinsic and extrinsic errors. Intrinsic errors occur from the system itself, such as the wheels not being the same size. Extrinsic errors occur because of the environment, such as wheel slippage or nonflat regions of the floor. SLAM and Path Planning

Rather than localize the robot using odometry, we used the Lidar to scan the environment and keep track of landmarks to determine its position. We used the A* (A-star) algorithm to plan a path through the environment. An extracted portion of my A* python code can be found below in the code section.

Competition Day

Although our robot was very good at getting to the goal without collisions, improvements can be made to speed by assigning multiple navigation points after every "mapping" step. Our implementation erred on the side of caution by checking for imminent wall collisions every step of the way, which drastically slowed the robot down.
